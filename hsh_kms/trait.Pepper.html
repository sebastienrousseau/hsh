Expand description

A pepper provider — produces an HMAC-SHA-256 tag over the password keyed by the secret material for a given KeyVersion.

Implementations must be Send + Sync so a Policy carrying a pepper can be shared across worker threads.

fn apply( &self, version: KeyVersion, password: &[u8], ) -> Result<[u8; 32], PepperError>

Computes HMAC-SHA-256(key_at(version), password) and returns the 32-byte tag. Errors if the requested version is not available in this provider.

§Errors

Returns PepperError::UnknownVersion if the version isn’t stored, or PepperError::Backend if the backend (KMS) fails.

Source

fn current(&self) -> KeyVersion

Returns the key version to use for new hashes. Older versions remain usable via Pepper::apply for verifying existing hashes; rotation is handled by hsh::api::verify_and_upgrade.
