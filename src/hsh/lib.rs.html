9//! # Hash (HSH) — multi-algorithm password hashing for Rust
10//!
11//! PHC-formatted hash storage with constant-time verification, KMS-backed
12//! peppering, FIPS 140-3 fail-closed contract, and automatic rehash on
13//! policy drift. Built on the RustCrypto stack with
14//! `#![forbid(unsafe_code)]` workspace-wide.
15//!
16//! [![Crates.io][crate-shield]](https://crates.io/crates/hsh)
17//! [![Docs.rs][docs-shield]](https://docs.rs/hsh)
18//! [![Lib.rs][lib-rs-shield]](https://lib.rs/crates/hsh)
19//! [![License][license-shield]](https://opensource.org/licenses/MIT)
20//! [![Rust][rust-shield]](https://www.rust-lang.org)
21//!
22//! ## Quick start
23//!
24//! ```rust
25//! use hsh::{Outcome, Policy, api};
26//!
27//! fn main() -> Result<(), hsh::Error> {
28//!     let policy = Policy::owasp_minimum_2025();
29//!     let stored = api::hash(&policy, "correct horse battery staple")?;
30//!
31//!     let outcome = api::verify_and_upgrade(
32//!         &policy,
33//!         "correct horse battery staple",
34//!         &stored,
35//!     )?;
36//!
37//!     assert!(outcome.is_valid());
38//!     assert!(!outcome.needs_rehash());
39//!     Ok(())
40//! }
41//! ```
42//!
43//! ## What `hsh` ships in v0.0.9
44//!
45//! | Algorithm | Status | OWASP-2025 default |
46//! | --------- | ------ | ------------------ |
47//! | **Argon2id** (default) | ✅ Recommended | `m = 19 456 KiB`, `t = 2`, `p = 1` |
48//! | **Bcrypt** | ✅ Hardened — 72-byte safety rail (CVE-2025-22228) | `cost = 10` |
49//! | **Scrypt** | ✅ Configurable | `N = 2^17`, `r = 8`, `p = 1` |
50//! | **PBKDF2-HMAC-SHA-256/512** | ✅ FIPS-eligible | `iters = 600 000` / `210 000` |
51//! | Argon2i / Argon2d | Verify-only (legacy) | — |
52//!
53//! The verifier accepts any of the four production algorithms above
54//! interchangeably (plus the legacy Argon2 variants); the live
55//! [`Policy`] only governs *new* hashes and rehash targets.
56//!
57//! ## What `hsh` is **not**
58//!
59//! - **Not post-quantum cryptography.** Memory-hard KDFs like Argon2id
60//!   raise the cost of offline brute-force on both classical and
61//!   quantum hardware (Grover yields only a √-speedup), but they are
62//!   not PQ primitives. For ML-KEM, ML-DSA, or SLH-DSA, use
63//!   [`aws-lc-rs`](https://crates.io/crates/aws-lc-rs).
64//! - **Not a self-validating FIPS 140-3 module.** The crate carries a
65//!   [`Backend::Fips140Required`] *contract* — [`api::hash`] refuses to
66//!   mint hashes outside FIPS-routed primitives — but the underlying
67//!   crypto today is the pure-Rust RustCrypto stack. The dedicated
68//!   `hsh-backend-awslc` follow-up routes through the validated
69//!   `aws-lc-rs` FIPS module without changing the public API. See
70//!   [`doc/FIPS.md`][fips-doc] and [`doc/adr/0004-fips-strategy.md`][adr-fips].
71//! - **Not a general-purpose digest library.** For SHA-2 / SHA-3 /
72//!   BLAKE3 content addressing, use the companion
73//!   [`hsh-digest`](https://crates.io/crates/hsh-digest) crate.
74//!
75//! ## Architecture
76//!
77//! - **Policy-driven**: a versioned [`Policy`] declares the primary
78//!   algorithm and per-algorithm parameters. Construct via the
79//!   [`Policy::owasp_minimum_2025`] / [`Policy::rfc9106_first_recommended`]
80//!   / [`Policy::fips_140_pbkdf2`] presets or the [`policy::PolicyBuilder`].
81//! - **Auto-rehash on drift**: [`api::verify_and_upgrade`] returns an
82//!   [`Outcome`] whose `Valid` variant folds the new PHC string into
83//!   `rehashed: Option<String>` — `Some(_)` precisely when the stored
84//!   hash falls below current policy. The caller persists the new value
85//!   on next login.
86//! - **Optional peppering**: with the `pepper` feature, [`Policy::with_pepper`]
87//!   attaches an HMAC-SHA-256 pepper provider with versioned key
88//!   rotation. Hashes carry a `hsh-pepper:<version>:` wrapper; rotation
89//!   is non-destructive.
90//! - **Structured errors**: [`Error`] is a `#[non_exhaustive]`
91//!   `thiserror` enum with `Clone + Send + Sync` and a typed
92//!   [`error::HashingErrorKind`] discriminant for downcasting without
93//!   parsing strings.
94//!
95//! ## Cargo features
96//!
97//! | Feature | Default | What it adds |
98//! | ------- | ------- | ------------ |
99//! | `pepper` | off | KMS-backed peppering via the `hsh-kms` companion crate |
100//! | `fips` | off | Forward-compat marker for the `aws-lc-rs` FIPS backend |
101//! | `compat-v0_0_x` | off | Re-exposes the pre-0.0.9 stringly-typed API for migration |

155/// Library entry point used by the `hsh` binary.
156///
157/// Prints a short welcome banner and returns `Ok(())`. Exists so the
158/// `hsh-cli` binary has a single library-side entry to call into.
159///
160/// # Errors
161///
162/// Returns [`Error::Verification`] only when the `HSH_TEST_MODE`
163/// environment variable is set to `"1"` — the integration test suite
164/// uses this to exercise the error-propagation path.
165pub fn run() -> Result<()> {
166    if std::env::var("HSH_TEST_MODE").unwrap_or_default() == "1" {
167        return Err(Error::Verification("simulated error".into()));
168    }
169
170    let name = "hsh";
171    println!("Welcome to `{}` 👋!", name.to_uppercase());
172    println!("Enterprise password hashing for Rust.");
173    Ok(())
174}
